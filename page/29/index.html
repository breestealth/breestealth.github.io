啤酒游戏反思

發表於 2014-03-13

昨天终于在部门完成了一次「啤酒游戏」。在这个过程中的确是学到了不少。

对于这个游戏就不做过多的解释，Google一下一大把。我只是简单的描述了一下这个游戏过程中遇到的一些事情（不完全是坏事）。

  1. 一开始讲解规则的时候，很多人都表示看不懂；
  2. 前两轮需要讲解很多，并且需要对每个字段的讲解，反复多次；
  3. 流程较多，导致作为司机的我，有时候会对订货单、发货单等等弄得混淆，导致了游戏部份时刻进展缓慢；
  4. 不按照常理出牌。这个是较为头疼的事情。有些零售商组在前4轮，就会有100甚至以上的定单，导致了其他零售商的库存不足，也导致了原本应该推迟出现的「定单暴增」明显提前，导致了后续的累加效应不明显，甚至到了16周时，制造商竟然还亏欠了较多的啤酒（约170箱）；
  5. 算术问题。有些组依然能将总定单数算错，导致了部份零售商会收到「比预期更多的啤酒」；
  6. 整个游戏过程，绝大多数同事还是兴致盎然，有部份同事觉得索然无味，甚至开始聊天、玩手机等。这点没有想到有什么办法可以杜绝；
  7. 最后的总结部份还算成功，将一个结果并不如预期的游戏总结成了一个还算OK的结果。起到的作用不好说，但至少有部份人可以听进去，希望有所改进。

在整个游戏的过程中，个人觉得还有优化的有以下几个地方：

  1. 人员座位。本次游戏，人员的座位做了2圈。里面的小圈坐的全部是零售商，外面坐了3个批发商和1个制造商。这种布局至少有以下几点问题：a）零售商靠得太近，难免会有语言沟通 b）我为了分发纸片，跑得路太多；
  2. 赌博行为。有些组看到第一周的需求增加了100％（4－>8），然后就产生了「赌一把」的想法，第二周就将订货数上升到了100箱。这种行为在游戏中应该极力避免。该游戏应该是以「这是我的事业」的心态去做，而不是「赌博」的心态。可以考虑对这种行为产生一定的惩罚，从而避免；
  3. 报市场需求的问题。每次都是用纸片的方式报告市场需求，每次均为8箱，这比较容易让人产生「猜测下一次市场需求」的行为。 个人考虑 是否可以随机的将整个市场需求进行分段，从而避免了每次的猜测行为。但是这也就违背了游戏设计之初的「需求是稳定」这个前提，所以该条待讨论；
  4. 聊天行为。这个没有非常好的办法，只能依赖座位来改进；
  5. 道具问题。本次游戏，所有的道具（定单）均为打印纸，这种纸片在大规模分发中非常不方便，而且前期制作困难，裁纸的工作量巨大。可以考虑相关定单，以「随手撕下纸片」的方式来处理，不需要所有的都如此尽善尽美，毕竟所有的定单的使用寿命也只有一次，可以考虑「用完就抛弃」；
  6. 人员问题。人情世故在这个游戏中还是暴露了。在缺货的情况下，批发商的发货方式就明显有偏向了，这点非常不好。下次可以考虑不同组之间身份互相不明朗（即只发送身份卡，但是不直接亮具体身份），这样子就可以避免这个问题的发生；
  7. 信息通知问题。原本应该在第7、8、10周分别告知制造商、零售商和批发商为什么啤酒销量大增的原因。由于我的失误，忘记告知了，所以该消息在第9周才直接公布给了三方人员；
  8. 时间控制问题。本次游戏从2:30PM开始到5:00PM，游戏只进行了16周，距离希望的30周还有一些距离。

大概总结了一下，这就是本次啤酒游戏的过程中需要注意以及个人对整体结果的反思。

PS：最终的结果，并没有大规模出现「所有人都大量囤货」的现象。个人感觉有以下几点导致了该情况的发生：

  1. 前期部份组破坏规则的赌博行为，过早的将第一次「需求高峰」向前推进了，从而使得「需求高峰」出现时间较早，并且后续的持续时间较长，使得批发商与生产商均有较长时间进行准备；
  2. 部份零售商通过需求变化规律，很早的就制定了相关的应对措施，使得情况有所好转；
  3. 很多零售商没有采用「不给我货，我就多定一些」这种策略；
  4. 因为很多人相互认识，所以在发货方面也有一些人情陷入其中，导致发货并非绝对客观和公正。

年度工作总结

發表於 2014-01-30

依然习惯用农历年来象征一年的结束，总感觉那样子才能给自己的工作做一个完整的归纳。

2013年，总体来说，个人的工作计划虽然有所实现，但是并没有完美达成。究其原因，还是在于关系的梳理、处理以及个人能力不足等方面，让我原先的计划有了较为明显的受阻。

总结了一下，今年个人完成工作中，最满意的工作有这么几件：

  1. Team Building。团建工作一直以来都是我工作的重点，今年也不例外。在这一年的工作中，的确是发掘了几个好苗子，有努力培养的潜质。明年还是要继续给予机会，交予更大的责任。而整个团队现在的凝聚力虽然没有达到我最初的想法，但是也的确比年初有所提高，这点也足以让我欣慰。另外为了明年可以将团队打造成「学习型团队」，我也做下了一定的伏笔，为明年工作做准备；
  2. 产品研发推进。今年由于部门有了一个产品经理，这一块有了长足的发展，但是依然不够。就我现在的观点来看，当前我们部门在产品研发方面依然很匮乏——甚至连产品研发都不是，仅仅是功能研发。在这种环境下，明年能成功迈入产品研发阶段也尤未可知，这也将是我明年最为紧张的一点；
  3. 规范化。从2013年年初开始推行Trello到2013年年末开始推行Jira，都是根据部门、公司当前的状态，适当的调整了相关的工具选择和使用，让整个团队的工作可以更加切实有效的开展起来；
  4. 个人能力方面。个人能力在2013年提升最大的是沟通协调能力。在部门内部、跨部门等沟通中，也的确做到了「更加有效的沟通」，这点算是我本年度最大的进步。至于项目管理的理论学习等方面，本年度进步有限，有点惭愧。在2014年，一定要参加一门其他的项目管理的考试，增强一下个人在这方面的能力；
  5. 行政方面。在行政方面，本年度也配合部门经理完成了「版本发布计划」、「绩效考核」等部门行政文件；
  6. 岗位晋升。这点算是我最不看中的一点。本年度，终于升职为「部门副经理」的岗位（原本给我Offer的时候就说了是副经理，最后给了是一个「经理助理」）。不过个人感觉并没有太多的变化，尤其是在我做事的职权以及责任方面，并没有太大的改变。另外，在本年度公司的体系改革中，也加入了相关的管理组。

今年工作中，我需要反思的依然有很多：

  1. 工作的深入程度不足。在研发同事辛辛苦苦工作的时候，我并没有非常深入的去了解工作，仅仅是在WBS甚至功能的层面进行任务分配，却并没有做最后的check的工作。这种作法导致了在数次的产品交付过程中的事故的发生。（此处有话说：正常的企业中，我的作法没有太大问题，但是在当前企业环境下，也就是「事业环境因素」中，我的作法会导致一些问题的发生。这点需要在日后的工作中，提升同事能力和意识同时，补充相应的岗位，让不同工种之间工作可以衔接的更加无缝，从而提升工作效率。）
  2. 沟通依然没有完全到位。对于与部门经理以及更高层面的领导，我的沟通依然不足，依然不能完全达到我的原始预期。造成这个原因有很多，包括公司现状、我的话术以及时机等等。但不论哪一点，都表现出我个人对这方面的把控不足；
  3. 员工的能力提升依然有限。也许在技术（Technology）层面，很多人都有进步，但是在技术之外的层面，包括责任心、沟通能力、领导能力、时间管理等方面，很多同事，尤其是组长级的同事，还是没有表现出其应该到达的高度。这点应该是我在给予指导方面有不足，这点明年还是需要进一步的给予协助；
  4. 产品质量依然较为低下。这里我用的算是比较客气的说法。当前虽然产品已经有了长足的进步，但是如果以一个挑剔甚至说较为中庸的眼光来看的话，产品质量依然只能打4分（满分10分）。当前依然存在代码混乱、速度慢、易用性差等各种问题。而这其中每一个问题，都不是可以瞬间解决的，都需要花很长时间。而公司能提供多少时间，这依然是个未知数；
  5. 个人能力提升有限。这点比较惭愧，这一年自己在项目管理中提升并不是很明显。虽然有众多原因可以解释，但是我个人依然难辞其咎。在下一年，这一块将是工作中个人能力提升的一个重点。

整体来看，今年一年，工作计划并没有完全完成，大概完成了80％，另外20％由于各种原因（事业环境因素、个人能力等）并没有完成。客观来说，这种完成度我已经没有什么要说的了，可以说与我的个人能力相匹配了。但是从我的岗位职责来说，我觉得依然不足，毕竟这与我给自己的「仆人」定位不完全吻合，我应该为下面的人做得更多更好。

无论怎么说，2013年过去了，2014年即将开始。成绩和不足都不再重要，重要的是要继续向着2014年的计划和目标努力。
